# 系統設計練習工具

一個 Electron 桌面應用程式，協助你練習撰寫系統規格書（SRS）、AI Skill 文件、系統設計文件，並提供即時評論與改善建議。內建情境題庫，讓你在接近真實面試或工作的情境下練習。

---

## 功能特色

- **三種文件類型**：SRS 規格書、AI Skill、系統設計文件，各有獨立評分標準
- **情境練習題庫**：每種文件類型各 3 道情境題，涵蓋初級／中級／高級難度，內附需求清單與撰寫提示
- **AI 即時出題**：啟用 Ollama 後可即時生成全新情境題目，題目不重複
- **規則式評論**：自動檢查必要章節是否存在、標記品質問題（模糊用詞、缺少數字指標等）
- **評分圓環**：直觀顯示完成度（0–100 分）與各項統計
- **本地 AI 模型**：可選接 Ollama 進行深度文字評論（需另行安裝）
- **多種輸入方式**：直接輸入、上傳 `.txt`/`.md` 檔案、拖曳上傳

## 使用步驟

### 步驟一：選擇文件類型

點擊頂部頁籤切換你要練習的文件類型：
- **SRS 規格書**：Software Requirements Specification，軟體需求規格書
- **AI Skill**：AI Agent 的技能描述文件
- **系統設計**：High-level 系統架構設計文件

### 步驟二（建議）：取得情境題目

點擊 header 的「📋 情境練習」按鈕，面板會在編輯器上方展開：

| 操作 | 說明 |
|------|------|
| **💡 提示** | 展開／收起撰寫提示，卡關時再看 |
| **↺ 換題** | 隨機換一道同類型的新題目 |
| **✨ AI 出題** | 呼叫本地 Ollama 即時生成全新題目（需啟用本地模型） |
| **✕** | 收起情境面板 |

切換頁籤時，情境面板會自動切換到該文件類型的題目。

> **難度說明**：★☆☆ 初級 / ★★☆ 中級 / ★★★ 高級

### 步驟三：輸入文件內容

有三種方式：
1. **直接輸入**：在左側編輯器直接打字或貼上內容
2. **上傳檔案**：點擊「上傳檔案」選擇 `.txt` 或 `.md` 檔案
3. **拖曳上傳**：將檔案直接拖曳到編輯器區域

### 步驟四：點擊「開始評論」

右側面板會顯示：

| 項目 | 說明 |
|------|------|
| **評分（0–100）** | 依已完成章節計算，70+ 良好、40–69 待改善、40 以下需加強 |
| **缺失項目** | 尚未涵蓋的章節，紅色標示為「必要」項目 |
| **品質警告** | 模糊用詞、缺少數字指標、未使用需求編號等 |
| **已完成項目** | 偵測到的章節清單 |

### 步驟五（選用）：啟用 AI 評論

1. 點擊右上角「設定」
2. 勾選「啟用本地模型」
3. 輸入 Ollama API 網址（預設 `http://localhost:11434`）和模型名稱（如 `llama3`）
4. 點擊「測試連線」確認可用
5. 儲存後，下次評論時會多出「AI 詳細評論」區塊，情境面板也會出現「✨ AI 出題」按鈕

## 情境題庫

每種文件類型各內建 3 道題目，涵蓋三種難度：

### SRS 規格書

| 難度 | 題目 | 重點 |
|------|------|------|
| ★☆☆ | 線上書店系統 | 基礎 CRUD、多角色、訂單流程 |
| ★★☆ | 外賣訂餐平台訂單模組 | 狀態機、即時推送、退款補償 |
| ★★★ | 企業請假暨差勤系統 | 複雜業務規則、多層審核、外部整合 |

### AI Skill

| 難度 | 題目 | 重點 |
|------|------|------|
| ★☆☆ | 客服問答機器人 | RAG、信心度、多輪對話 |
| ★★☆ | 智慧合約審閱助手 | 文件前處理、風險分類、反饋學習 |
| ★★★ | 多代理人資料分析系統 | NL2SQL、多 Agent 協作、安全模型 |

### 系統設計

| 難度 | 題目 | 重點 |
|------|------|------|
| ★☆☆ | URL 縮短服務 | 短碼生成、快取策略、規模估算 |
| ★★☆ | 即時聊天系統 | WebSocket、離線訊息、媒體儲存 |
| ★★★ | 全球影片串流平台 | 轉碼 pipeline、CDN、DRM |

## 評分標準說明

### SRS 規格書（13 項）

| 章節 | 必要 | 權重 |
|------|------|------|
| 文件目的 | ✓ | 8 |
| 系統範圍 | ✓ | 8 |
| 系統概述 | ✓ | 8 |
| 功能需求 | ✓ | 15 |
| 非功能需求 | ✓ | 12 |
| 效能需求 | ✓ | 10 |
| 安全需求 | ✓ | 10 |
| 定義與縮寫 | - | 5 |
| 使用者描述 | - | 5 |
| 系統限制 | - | 5 |
| 外部介面需求 | - | 7 |
| 使用案例 | - | 8 |
| 驗收標準 | - | 7 |

品質警告：模糊用詞、缺少需求編號（FR-001）、缺少量化指標（ms/QPS）、缺少優先級標記

### AI Skill 文件（11 項）

必要項目：Skill 名稱、功能描述、觸發條件、輸入參數、輸出格式、使用範例、錯誤處理、限制說明

品質警告：缺少參數型別、缺少必填/選填標記、缺少邊界情況說明

### 系統設計文件（14 項）

必要項目：系統概述、需求分析、系統架構、元件說明、資料流設計、資料庫設計、可擴展性設計、安全設計、技術選型

品質警告：缺少規模數字估算（QPS/DAU）、缺少設計取捨說明、缺少瓶頸分析

## 本地模型設定（Ollama）

```bash
# 安裝 Ollama（官網下載後）
ollama pull llama3        # 下載 Llama 3 模型（約 4GB）
ollama pull mistral       # 或使用 Mistral

# 啟動 Ollama 服務（通常開機自動啟動）
ollama serve
```

設定畫面中輸入：
- API 網址：`http://localhost:11434`
- 模型名稱：`llama3`（或你已下載的模型名稱）

