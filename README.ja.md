# システム設計練習ツール

SRS 仕様書・AI Skill ドキュメント・システム設計ドキュメントの執筆練習をサポートする Electron デスクトップアプリです。リアルタイムの評価とフィードバックを提供します。内蔵のシナリオ問題ライブラリを搭載し、繁体字中国語・英語・日本語に対応しています。

---

## 機能

- **3 種類のドキュメントタイプ**：SRS 仕様書・AI Skill・システム設計（各タイプに独自の評価ルール）
- **シナリオ練習ライブラリ**：各ドキュメントタイプに 3 問（初級・中級・上級）
- **AI シナリオ生成**：Ollama を有効化すると、新しいシナリオ問題をその場で生成できる
- **ルールベース評価**：必要なセクションの有無を自動確認、品質問題（曖昧な表現・指標の欠如など）をフラグ
- **スコアリング**：完成度（0〜100）と完了・不足・警告の内訳をビジュアル表示
- **ローカル AI モデル**：Ollama との連携で詳細なテキストレビューが可能（オプション）
- **複数の入力方法**：直接入力・`.txt`/`.md` ファイルのアップロード・ドラッグ＆ドロップ
- **多言語 UI**：ヘッダーで繁体字中国語・英語・日本語を切り替え可能

## 使い方

### ステップ 1：ドキュメントタイプを選択

上部のタブをクリックして切り替えます：
- **SRS 仕様書**：Software Requirements Specification（ソフトウェア要件仕様書）
- **AI Skill**：AI エージェントのスキル説明ドキュメント
- **システム設計**：ハイレベルなシステムアーキテクチャ設計ドキュメント

### ステップ 2（推奨）：シナリオ問題を取得

ヘッダーの **📋 シナリオ** ボタンをクリックすると、エディタの上にパネルが展開されます：

| 操作 | 説明 |
|------|------|
| **💡 ヒント** | 記述のヒントを表示/非表示（行き詰まったときだけ確認） |
| **↺ 次の問題** | 同じドキュメントタイプのランダムな問題に切り替え |
| **✨ AI 出題** | ローカル Ollama を呼び出して新しい問題を即時生成（ローカルモデルが必要） |
| **✕** | シナリオパネルを閉じる |

タブを切り替えると、そのドキュメントタイプのシナリオが自動的に読み込まれます。

> **難易度**：★☆☆ 初級 / ★★☆ 中級 / ★★★ 上級

### ステップ 3：ドキュメントを入力

3 種類の入力方法があります：
1. **直接入力**：左側のエディタに直接入力またはペースト
2. **ファイルをアップロード**：「ファイルを開く」をクリックして `.txt` または `.md` ファイルを選択
3. **ドラッグ＆ドロップ**：ファイルをエディタエリアにドロップ

### ステップ 4：「評価する」をクリック

右側のパネルに以下が表示されます：

| 項目 | 説明 |
|------|------|
| **スコア（0〜100）** | 完了セクションに基づいて計算：70+ 良好・40〜69 要改善・40 未満 要強化 |
| **不足項目** | まだカバーされていないセクション（赤いバッジ = 必須） |
| **警告** | 曖昧な表現・数値指標の欠如・要件 ID なし など |
| **完了項目** | 検出されたセクションの一覧 |

### ステップ 5（オプション）：AI レビューを有効化

1. ヘッダーの **⚙ 設定** をクリック
2. **「ローカルモデル（Ollama）を使用した AI レビューを有効化」** にチェックを入れる
3. Ollama API URL（デフォルト `http://localhost:11434`）とモデル名（例：`llama3`）を入力
4. **「接続テスト」** で動作確認
5. 設定を保存後、次回の評価から **「AI 詳細レビュー」** セクションが追加されます。シナリオパネルには **「✨ AI 出題」** ボタンも表示されます。

## シナリオライブラリ

各ドキュメントタイプに 3 問の内蔵シナリオがあります（3 段階の難易度）：

### SRS 仕様書

| 難易度 | シナリオ | フォーカスエリア |
|--------|----------|-----------------|
| ★☆☆ | オンライン書店システム | 基本的な CRUD・複数ロール・注文フロー |
| ★★☆ | フードデリバリー注文モジュール | ステートマシン・リアルタイムプッシュ・返金補償 |
| ★★★ | 企業休暇・勤怠管理システム | 複雑なビジネスルール・多段階承認・外部 API 連携 |

### AI Skill

| 難易度 | シナリオ | フォーカスエリア |
|--------|----------|-----------------|
| ★☆☆ | カスタマーサービス Q&A ボット | RAG・信頼度スコア・マルチターン会話 |
| ★★☆ | スマート契約審査アシスタント | ドキュメントパイプライン・リスク分類・フィードバックループ |
| ★★★ | マルチエージェントデータ分析システム | NL2SQL・マルチエージェント協調・セキュリティモデル |

### システム設計

| 難易度 | シナリオ | フォーカスエリア |
|--------|----------|-----------------|
| ★☆☆ | URL 短縮サービス | 短縮コード生成・キャッシュ・キャパシティ推定 |
| ★★☆ | リアルタイムチャットシステム | WebSocket・オフラインメッセージ・メディアストレージ |
| ★★★ | グローバル動画ストリーミングプラットフォーム | トランスコードパイプライン・CDN・DRM |

## 評価基準

### SRS 仕様書（13 セクション）

| セクション | 必須 | 重み |
|-----------|------|------|
| 目的 | ✓ | 8 |
| スコープ | ✓ | 8 |
| システム概要 | ✓ | 8 |
| 機能要件 | ✓ | 15 |
| 非機能要件 | ✓ | 12 |
| 性能要件 | ✓ | 10 |
| セキュリティ要件 | ✓ | 10 |
| 定義と略語 | - | 5 |
| ユーザー説明 | - | 5 |
| システム制約 | - | 5 |
| 外部インターフェース要件 | - | 7 |
| ユースケース | - | 8 |
| 受入基準 | - | 7 |

品質警告：曖昧な表現・要件 ID（FR-001）の欠如・定量指標（ms/QPS）の欠如・優先度ラベルの欠如

### AI Skill ドキュメント（11 セクション）

必須項目：Skill 名・機能説明・トリガー条件・入力パラメータ・出力フォーマット・使用例・エラー処理・制限事項

品質警告：パラメータ型の欠如・必須/オプションの明示なし・エッジケースの説明なし

### システム設計ドキュメント（14 セクション）

必須項目：システム概要・要件分析・アーキテクチャ・コンポーネント説明・データフロー・データベース設計・スケーラビリティ設計・セキュリティ設計・技術スタック

品質警告：規模推定（QPS/DAU）の欠如・設計トレードオフの説明なし・ボトルネック分析なし

## ローカルモデルのセットアップ（Ollama）

```bash
# 公式サイトから Ollama をインストール後：
ollama pull llama3        # Llama 3 モデルをダウンロード（約 4GB）
ollama pull mistral       # または Mistral を使用

# Ollama サービスを起動（通常は起動時に自動起動）
ollama serve
```

設定パネルで入力：
- API URL：`http://localhost:11434`
- モデル名：`llama3`（またはダウンロードしたモデル名）

```
    ├── style.css        # ダークテーマスタイル
    ├── locales.js       # UI 文字列（zh-TW / en / ja）

```
