# システム設計練習ツール

SRS 仕様書・AI Skill ドキュメント・システム設計ドキュメントの執筆練習をサポートする Electron デスクトップアプリです。リアルタイムの評価とフィードバックを提供します。内蔵のシナリオ問題ライブラリを搭載し、繁体字中国語・英語・日本語に対応しています。

---

## 機能

- **3 種類のドキュメントタイプ**：SRS 仕様書・AI Skill・システム設計（各タイプに独自の評価ルール）
- **シナリオ練習ライブラリ**：各ドキュメントタイプに 43 問（初級・中級・上級）
- **AI シナリオ生成**：Ollama を有効化すると、新しいシナリオ問題をその場で生成できる
- **ルールベース評価**：必要なセクションの有無を自動確認、品質問題（曖昧な表現・指標の欠如など）をフラグ
- **スコアリング**：完成度（0〜100）と完了・不足・警告の内訳をビジュアル表示
- **ローカル AI モデル**：Ollama との連携で詳細なテキストレビューが可能（オプション）
- **複数の入力方法**：直接入力・`.txt`/`.md` ファイルのアップロード・ドラッグ＆ドロップ
- **多言語 UI**：ヘッダーで繁体字中国語・英語・日本語を切り替え可能

## 使い方

### ステップ 1：ドキュメントタイプを選択

上部のタブをクリックして切り替えます：
- **SRS 仕様書**：Software Requirements Specification（ソフトウェア要件仕様書）
- **AI Skill**：AI エージェントのスキル説明ドキュメント
- **システム設計**：ハイレベルなシステムアーキテクチャ設計ドキュメント

### ステップ 2（推奨）：シナリオ問題を取得

ヘッダーの **📋 シナリオ** ボタンをクリックすると、エディタの上にパネルが展開されます：

| 操作 | 説明 |
|------|------|
| **💡 ヒント** | 記述のヒントを表示/非表示（行き詰まったときだけ確認） |
| **↺ 次の問題** | 同じドキュメントタイプのランダムな問題に切り替え |
| **✨ AI 出題** | ローカル Ollama を呼び出して新しい問題を即時生成（ローカルモデルが必要） |
| **✕** | シナリオパネルを閉じる |

タブを切り替えると、そのドキュメントタイプのシナリオが自動的に読み込まれます。

> **難易度**：★☆☆ 初級 / ★★☆ 中級 / ★★★ 上級

### ステップ 3：ドキュメントを入力

3 種類の入力方法があります：
1. **直接入力**：左側のエディタに直接入力またはペースト
2. **ファイルをアップロード**：「ファイルを開く」をクリックして `.txt` または `.md` ファイルを選択
3. **ドラッグ＆ドロップ**：ファイルをエディタエリアにドロップ

### ステップ 4：「評価する」をクリック

右側のパネルに以下が表示されます：

| 項目 | 説明 |
|------|------|
| **スコア（0〜100）** | 完了セクションに基づいて計算：70+ 良好・40〜69 要改善・40 未満 要強化 |
| **不足項目** | まだカバーされていないセクション（赤いバッジ = 必須） |
| **警告** | 曖昧な表現・数値指標の欠如・要件 ID なし など |
| **完了項目** | 検出されたセクションの一覧 |

### ステップ 5（オプション）：AI レビューを有効化

1. ヘッダーの **⚙ 設定** をクリック
2. **「ローカルモデル（Ollama）を使用した AI レビューを有効化」** にチェックを入れる
3. Ollama API URL（デフォルト `http://localhost:11434`）とモデル名（例：`llama3`）を入力
4. **「接続テスト」** で動作確認
5. 設定を保存後、次回の評価から **「AI 詳細レビュー」** セクションが追加されます。シナリオパネルには **「✨ AI 出題」** ボタンも表示されます。

## シナリオライブラリ

各ドキュメントタイプに **43 問の内蔵シナリオ**（3 問オリジナル + 40 問の実践的新問題）があります（3 段階の難易度）：

### SRS 仕様書（43 問）

| 難易度 | シナリオ例 |
|--------|-----------|
| ★☆☆ | オンライン書店システム・EC 商品レビュー・HR 従業員オンボーディング… |
| ★★☆ | フードデリバリー注文モジュール・マルチプラットフォーム在庫管理・オンライン教育ライブクラス… |
| ★★★ | 企業休暇・勤怠管理・金融帳票照合・越境 EC コンプライアンス… |

### AI Skill（43 問）

| 難易度 | シナリオ例 |
|--------|-----------|
| ★☆☆ | カスタマーサービス Q&A ボット・画像コンテンツ自動タグ付け・社内メール自動分類… |
| ★★☆ | スマート契約審査アシスタント・ブランドセンチメント監視・SEO コンテンツ改善アドバイザー… |
| ★★★ | マルチエージェントデータ分析・EC 検索リランキングエンジン・リアルタイム不正検出… |

### システム設計（43 問）

| 難易度 | シナリオ例 |
|--------|-----------|
| ★☆☆ | URL 短縮サービス・ショッピングカートシステム・ゲームリーダーボード… |
| ★★☆ | リアルタイムチャットシステム・プッシュ通知システム・電子決済送金… |
| ★★★ | グローバル動画ストリーミング・オンライン共同ドキュメント編集・EC フラッシュセール… |

## 評価基準

### SRS 仕様書（13 セクション）

| セクション | 必須 | 重み |
|-----------|------|------|
| 目的 | ✓ | 8 |
| スコープ | ✓ | 8 |
| システム概要 | ✓ | 8 |
| 機能要件 | ✓ | 15 |
| 非機能要件 | ✓ | 12 |
| 性能要件 | ✓ | 10 |
| セキュリティ要件 | ✓ | 10 |
| 定義と略語 | - | 5 |
| ユーザー説明 | - | 5 |
| システム制約 | - | 5 |
| 外部インターフェース要件 | - | 7 |
| ユースケース | - | 8 |
| 受入基準 | - | 7 |

品質警告：曖昧な表現・要件 ID（FR-001）の欠如・定量指標（ms/QPS）の欠如・優先度ラベルの欠如

### AI Skill ドキュメント（11 セクション）

必須項目：Skill 名・機能説明・トリガー条件・入力パラメータ・出力フォーマット・使用例・エラー処理・制限事項

品質警告：パラメータ型の欠如・必須/オプションの明示なし・エッジケースの説明なし

### システム設計ドキュメント（14 セクション）

必須項目：システム概要・要件分析・アーキテクチャ・コンポーネント説明・データフロー・データベース設計・スケーラビリティ設計・セキュリティ設計・技術スタック

品質警告：規模推定（QPS/DAU）の欠如・設計トレードオフの説明なし・ボトルネック分析なし

## ローカルモデルのセットアップ（Ollama）

```bash
# 公式サイトから Ollama をインストール後：
ollama pull llama3        # Llama 3 モデルをダウンロード（約 4GB）
ollama pull mistral       # または Mistral を使用

# Ollama サービスを起動（通常は起動時に自動起動）
ollama serve
```

設定パネルで入力：
- API URL：`http://localhost:11434`
- モデル名：`llama3`（またはダウンロードしたモデル名）

## 自動テスト

本プロジェクトは **Jest** を使用した自動テストを備えています。**137 個のテスト**で、ユニットテストと統合テストを網羅しています。

### テストの実行

```bash
# すべてのテストを実行
npm test

# ユニットテストのみ
npm run test:unit

# 統合テストのみ
npm run test:integration
```

### テスト構成

```
lib/
└── engine.js                      # テスト可能な純粋関数モジュール（evaluateWithRules、parseAIScenario）
tests/
├── unit/
│   ├── evaluateWithRules.test.js  # 評価エンジンのユニットテスト（19 件）
│   └── parseAIScenario.test.js    # AI シナリオ解析のユニットテスト（19 件）
└── integration/
    ├── rules.test.js              # ルール JSON 構造の統合テスト（47 件）
    ├── scenarios.test.js          # シナリオライブラリの統合テスト（34 件）
    └── ipcHandlers.test.js        # IPC ハンドラーのファイル I/O 統合テスト（18 件）
```

### テストカバレッジ

| 種別 | テスト対象 | カバー内容 |
|------|-----------|-----------|
| ユニット | `evaluateWithRules` | セクションキーワード検出・スコア/パーセンテージ計算・qualityChecks の双方向トリガーロジック |
| ユニット | `parseAIScenario` | 三言語（zh-TW / en / ja）解析・難易度マッピング・リスト解析・フォールバック動作 |
| 統合 | `rules/*.json` | フィールド構造・型・正規表現の正当性・maxScore 合計・評価エンジンとの連動 |
| 統合 | `scenarios/scenarios.json` | 129 問の数量・ID 一意性・三言語必須フィールド・要件/ヒントの最低数 |
| 統合 | IPC ファイル I/O | ルール・シナリオファイルの読み取り・設定の保存と読み込みのラウンドトリップ |
